Opis aplikacji Taskly
Szymon Piotrzkowski (analityk)


1. Informacje ogólne

Nazwa: Taskly
Typ: aplikacja webowa - menedżer zadań
Technologie: HTML, Tailwind CSS, JavaScript, Vue 3, GSAP, LocalStorage

Taskly to prosta aplikacja do organizowania codziennych zadań. Można w niej
tworzyć nowe wpisy, oznaczać je jako wykonane, edytować i usuwać te, które nie
są już potrzebne. Dodatkowo zadania da się filtrować po statusie, kategorii
i terminie.


2. Po co to powstało

Chcieliśmy zrobić proste narzędzie, które pomaga zaplanować dzień i nie
zgubić się w obowiązkach. Filtry mają ułatwić szybkie odnalezienie konkretnych
zadań, żeby użytkownik mógł skupić się na tym, co akurat ważne.


3. Funkcje

Zarządzanie zadaniami:
- dodawanie nowych zadań,
- edycja istniejących,
- usuwanie wybranych wpisów,
- oznaczanie jako wykonane.

Kategorie - każde zadanie należy do jednej z dwóch:
- "To-Do" - zwykłe, jednorazowe zadania,
- "Codzienne (Daily)" - rzeczy powtarzalne, wykonywane regularnie.

Terminy - do zadania można przypisać konkretną datę i godzinę.

Filtrowanie listy według:
- statusu (wszystkie / aktywne / ukończone),
- kategorii,
- terminu (zaległe / na dzisiaj / przyszłe).

Przechowywanie danych - wszystko zapisuje się automatycznie w LocalStorage,
więc po odświeżeniu strony lista zostaje na miejscu.


4. Interfejs

Na górze jest nagłówek z nazwą aplikacji i krótkim opisem, do czego służy.

Po lewej stronie znajduje się formularz tworzenia i edycji. Można w nim wpisać
nazwę zadania, wybrać kategorię, ustawić termin oraz zapisać zmiany albo
anulować edycję. Jeśli wymagane pole zostanie puste, pojawia się komunikat
o błędzie.

Po prawej wyświetla się lista zapisanych zadań. Przy każdym z nich widać
checkbox do zmiany statusu, nazwę, kategorię, termin oraz przyciski edycji
i usunięcia.


5. Jak to jest zbudowane

Projekt dzieli się na dwie główne części.

Plik HTML odpowiada za strukturę strony - rozmieszczenie formularza, listy
zadań i filtrów. Do stylowania użyliśmy Tailwind CSS, a dynamiczne elementy
obsługują dyrektywy Vue (v-model, v-if, v-for).

Plik app.js zawiera całą logikę: obsługę formularzy, filtrowanie, zapis
i odczyt z LocalStorage oraz animacje.


6. Najważniejsze funkcje w kodzie

loadTasks() - odczytuje zapisane dane z LocalStorage.

  function loadTasks() {
    const raw = localStorage.getItem(STORAGE_KEY)
    return raw ? JSON.parse(raw) : []
  }

saveTasks() - zapisuje aktualny stan do przeglądarki.

  function saveTasks(tasks) {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(tasks))
  }

submit() - obsługuje dodawanie i edycję zadań, po drodze sprawdza poprawność
wprowadzonych danych.

toggleDone() - zmienia status wybranego zadania.

  function toggleDone(id) {
    task.done = !task.done
  }

deleteTask() - usuwa wskazane zadanie z listy.

filteredTasks() - funkcja typu computed, która sama odświeża widok po zmianie
filtrów.


7. Wykorzystane technologie

HTML          - struktura strony
Tailwind CSS  - stylowanie
Vue 3         - logika aplikacji
GSAP          - animacje
LocalStorage  - zapisywanie danych


8. Animacje

Animacje dodaliśmy po to, żeby korzystanie z aplikacji było przyjemniejsze:
- elementy pojawiają się stopniowo po uruchomieniu,
- dodawanie i usuwanie zadań jest animowane,
- przy błędnym formularzu pojawia się efekt "shake",
- tryb edycji jest wyróżniany.

Do animacji użyliśmy biblioteki GSAP oraz wbudowanych przejść Vue.
